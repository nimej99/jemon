# 이미지 → 레이아웃 매핑 레시피

> 레퍼런스 이미지/도면(또는 요구사항 서술)을 받았을 때 코드 치기 전에 **반드시** 수행하는 분석 절차.
> 산출물(리전 맵 + 컴포넌트 매핑표 + 바인딩표)이 곧 구현 명세다. 절차 생략 = 퀄리티 복불복.

## 절차 (5단계)

### 1단계 — 리전 분해
이미지를 보고 ASCII 리전 맵을 그린다. 각 리전에 이름·대략 크기·역할.
- 질문: 히어로가 뭔가? (씬/지도/큰 차트) 사이드패널? KPI 스트립 위치?
- 히어로가 없으면 만들어라 — 고퀄 대시보드에는 항상 하나의 지배적 시각 요소가 있다.

### 2단계 — 요소 → 컴포넌트 매핑표
리전 안 모든 시각 요소를 `component-map.md` 컴포넌트로 매핑. 매핑 불가 요소 발견 → component-map.md에 신규 항목 정의부터.

### 3단계 — 데이터 바인딩표
요소마다 카탈로그 메트릭 키·집계·주기·포맷터. **바인딩 없는 시각 요소는 그리지 않는다.** 시연이면 mock 훅 + 시드 고정.

### 4단계 — 토큰 대조
이미지 색을 design-tokens.md 팔레트로 스냅. 팔레트 밖 색은 팔레트 쪽으로 보정(이미지를 따라가지 말 것).

### 5단계 — 셀프리뷰 예약
구현 후 quality-checklist.md 전 항목 + design-shot 스크린샷 육안 대조를 계획에 포함.

## 공통 골격 (전 레시피)

`AppShell(NavRail+TopBar)` → `PageHeader+LastUpdated` → **Hero**(화면 높이 ~65%) + 우측 SidePanel(~300px: DonutSummary+AlertList) → 하단 KPI Strip(StatCard×4~5+Sparkline).

## 레시피 A — 캠퍼스/사이트 조감도형 (골든 레퍼런스1)

**적용**: 캠퍼스/사옥/공장 부지 등 "공간에 분산된 장비" 모니터링.
1. Hero = `IsoCampusScene`: 건물 수 = 자산 그룹 수, kind로 형태 구별(강의동/도서관/체육관/기숙사/IT센터…). 야간 라이팅+Bloom, 아일랜드 지형(도로·트랙·분수·연못·태양광·수목).
2. `links` = 토폴로지(코어→건물). 흐름 방향 = 트래픽 방향.
3. 건물마다 오버레이 카드 — CPU/MEM/TRAFFIC/TEMP + 접속자. `cardAnchor`로 가장자리 분산, 겹침 금지.
4. KPI Strip: 전체 CPU/MEM/총트래픽/평균온도/총접속자.
5. 인터랙션: 카드/건물 호버 상호 하이라이트, 클릭 → 장비 상세.

## 레시피 B — 서버실 랙 실장도형 (레퍼런스2)

**적용**: 전산실/IDC 랙 단위 모니터링.
1. Hero = `RackElevation`: 랙 열 배치(통로 포함), U단위 슬롯, 장비 상태 = 전면 emissive.
2. 오버레이 = 랙별 요약(전력/온도/사용 U). 환경 센서(항온항습/UPS/누수)는 바닥·벽 노드.
3. SidePanel = 전력·온도 TimeSeries + AlertList. KPI = 총전력/평균온도/가동률/가용 U.
4. 온도 임계는 전산실 기준(warn 27 / crit 32).

## 레시피 C — 지리 분산형

**적용**: 다중 사이트/지점 WAN. Hero = GeoMap(MapLibre 다크 + deck.gl Arc=회선, Scatter=사이트). 나머지 A 골격. (GeoMap 최초 사용 시 component-map 갱신+구현 선행)

## 레시피 D — 운영 상세형 (씬 없음)

**적용**: 단일 도메인 심층(DB 성능 등). Hero = 대형 TimeSeries(멀티 시리즈)+Heatmap. 좌 TopN, 우 AlertList+테이블. 씬이 없어도 토큰·밀도 규칙 동일.

## 레시피 E — 아파트 단지형

**적용**: 공동주택 단지 시설 모니터링(동별 전력/급수/승강기/공용부 온도/세대 IoT).
1. Hero = `IsoCampusScene`, `kind:"apartment"` 타워 반복 + 놀이터/주차장/관리동. 배치는 단지 배치도를 따른다.
2. 오버레이 카드 지표: 전력(kW→MetricBar fillPct=부하율), 급수, 승강기 상태, 공용부 온도. metrics 계약이 안 맞으면 카드 커스텀 대신 **카탈로그 키→기존 4행에 매핑**하고 라벨만 교체.
3. links = 동↔관리동(통신/전력 계통). KPI = 단지 총전력/평균온도/민원(알림)/재실.

## 레시피 F — IoT 공장형

**적용**: 생산라인/설비 IoT(진동·온도·전류·가동률).
1. Hero = `IsoCampusScene`, `kind:"factory"`(톱니지붕)+창고+유틸리티동. 라인이 실내면 평면 배치를 부지 조감도로 승격.
2. 오버레이: 가동률(OEE)/전류/진동/온도. 설비 다운 = severity crit → 카드·노드·링크 시맨틱 전환.
3. links = 공정 흐름(원자재→라인→출하). 흐름 애니메이션 = 생산 흐름. KPI = OEE/생산량/에너지/불량률/가동 설비 수.

## 레시피 G — IoT 홈형 (스마트홈 온도 등)

**적용**: 주택/세대 단위 센서(온습도·전력·보안).
1. 소수 노드(방 3~8개)이므로 Hero는 단독주택 1채 아이소메트릭(`kind:"house"` + 방 단위 존) 또는 2D 평면도 SVG 폴백 — 노드가 적을 때 3D가 빈약하면 **평면도+글로우 존**이 더 고퀄.
2. 오버레이: 온도/습도/전력/재실. 임계는 쾌적 범위(예: 온도 warn <18 or >26 — 상한·하한 모두, severityOf 두 번 조합).
3. KPI = 평균온도/총전력/열림 센서/알림. SidePanel = 24h 온도 TimeSeries + AlertList.

## 공통 함정 (이거 하면 싸구려 된다)

- 흰 배경/기본 Tailwind 회색 카드 → 즉시 불합격.
- 차트 기본 테마(기본 팔레트, 밝은 그리드) 노출 → echartsBase 필수.
- 지표 라벨을 본문 크기로 → micro uppercase가 밀도의 핵심.
- 숫자 proportional 폰트 → tabular-nums.
- 오버레이 카드 완전 불투명 or 완전 유리 → 92% + blur가 정답.
- 씬을 이미지/일러스트로 대체 → 데이터 바인딩 불가. 반드시 실제 3D/SVG.
- 빈 상태 방치 → 상태 문구/스켈레톤 필수.
- 1440/1920 두 브레이크포인트 검증 생략 → 레이아웃 붕괴.
