# 합격 체크리스트 — 레퍼런스 패리티 게이트

> 대시보드 생성/수정 작업은 아래 전 항목 PASS + 스크린샷 육안 대조 없이는 "완료"라고 말할 수 없다.
> 방법: `node scripts/design-shot.mjs /<slug>` → 1440×900·1920×1080 스크린샷 → 골든 레퍼런스와 나란히 비교 → 항목별 PASS/FAIL 표를 결과에 첨부.

## A. 레이아웃 (구조 패리티)
- [ ] 리전 맵(1단계 산출물)과 실제 렌더의 리전 구성·비율이 일치한다.
- [ ] 히어로(씬/지도/대형차트)가 존재하고 화면의 지배적 요소다.
- [ ] 1440px과 1920px에서 붕괴·오버플로·겹침이 없다.
- [ ] 오버레이 카드가 서로/씬 주요 오브젝트와 겹치지 않고, 앵커가 올바른 대상을 가리킨다.

## B. 토큰 (디자인 시스템 준수)
- [ ] 모든 색이 tokens.ts/palette.ts 경유 — hex 리터럴 grep 시 토큰 정의 블록 외 0건.
- [ ] 다크 surface 위계(page < card < elevated)가 구분된다.
- [ ] 보더 전부 1px border-subtle. 시맨틱 컬러는 상태 표현에만.
- [ ] 타이포 스케일 준수, 지표 라벨은 micro uppercase.
- [ ] 모든 숫자 tabular-nums + 공통 포맷터 경유.

## C. 컴포넌트·차트
- [ ] 모든 시각 요소가 component-map.md의 컴포넌트다 (무단 신규 패턴 없음).
- [ ] 차트에 라이브러리 기본 테마 흔적 없음 (기본 팔레트/밝은 그리드/기본 툴팁).
- [ ] 스파크라인 축·범례·그리드 제거 + gradient fill.
- [ ] 도넛 중앙 총계·범례 카운트 = 세그먼트 합.

## D. 3D/씬 (해당 시)
- [ ] 아이소메트릭 카메라 각·야간 라이팅·Bloom이 레퍼런스 무드와 일치.
- [ ] 네트워크 라인/노드가 실제 토폴로지·상태 데이터에 바인딩 (장식 이펙트 0건).
- [ ] frameloop demand/인스턴싱, draw call ≤300.
- [ ] 노드 30개 초과 또는 WebGL 불가 시 2D 폴백 동작.

## E. 데이터
- [ ] 모든 표시 값이 훅/props 경유 — 컴포넌트 내 fetch/하드코딩 값 0건.
- [ ] 실데이터 모드에서 metric API 응답이 반영된다 (mock 시연이면 시드 고정 + 명시).
- [ ] 임계 초과 시 해당 지표 바/카드/노드가 시맨틱 컬러로 전환 (합성 초과값 주입으로 확인).
- [ ] LastUpdated가 실제 갱신 시각을 표시하고 주기 갱신된다.
- [ ] 빈 상태·로딩(스켈레톤)·에러 상태가 모두 디자인되어 있다.

## F. 인터랙션·접근성
- [ ] 호버/포커스 상태 정의 (카드 elevate, nav 하이라이트).
- [ ] 씬↔카드 상호 하이라이트 동작.
- [ ] prefers-reduced-motion에서 애니메이션 정지.
- [ ] 텍스트 대비 WCAG AA 근접 (muted 텍스트 포함).

## G. 빌드·위생
- [ ] `pnpm --filter @jemon/ui build` + `pnpm --filter @jemon/web typecheck` 0 에러.
- [ ] 새 런타임 의존성이 허용 라이선스(MIT/Apache-2.0/BSD/MPL-2.0/ISC)뿐 — `pnpm check:licenses` 통과.
- [ ] debug console.log 0건, 스텁/TODO 구현 0건.

## H. (선택) 회귀
- [ ] 골든 스크린샷 대비 픽셀 diff < 임계 (Playwright toHaveScreenshot).
